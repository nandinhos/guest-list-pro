Como estruturar a gestão de excursionistas?

Três abordagens diferentes — cada uma com trade-offs de UX e complexidade

A

Tabelas com ações inline + modais

Mantém o layout atual de relatório. Cada linha ganha botões Editar e Excluir. Um botão + Novo em cada seção abre um modal de cadastro. Tudo acontece sem sair da página.

Prós

  • Fluxo rápido, sem navegação
  • Contexto visual permanece
  • Leve de implementar

Contras

  • Formulários de modal têm espaço limitado
  • Sem ordenação/busca nas colunas
B

Página com abas — tabela Filament por entidade

Três abas: Excursões, Veículos, Monitores. Cada aba é uma tabela Filament nativa com busca, ordenação, filtros e ações completas (criar, editar, excluir). Modais de formulário do próprio Filament.

Prós

  • Busca e ordenação por coluna
  • Formulários espaçosos nos modais
  • Padrão Filament nativo — consistente

Contras

  • Perde a visão simultânea das 3 entidades
  • Mais código para configurar
C

Painel de gestão hierárquico

Seleciona uma Excursão na lista principal. Ao clicar, expande inline mostrando seus Veículos e, dentro de cada veículo, seus Monitores. Botões de ação contextuais em cada nível da hierarquia.

Prós

  • Reflete a hierarquia real dos dados
  • Navegação intuitiva Excursão → Veículo → Monitor
  • Evita confusão de qual veículo pertence a qual excursão

Contras

  • Mais complexo de implementar
  • Difícil buscar monitores sem saber a excursão
